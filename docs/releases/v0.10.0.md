# TraceHawk v0.10.0

TraceHawk v0.10.0 adds a privacy-bounded anonymous demo and an integrated cybersecurity learning
experience while preserving the protected local/private investigation workflow.

## Public Demo Safety

- Anonymous analysis accepts one bounded UTF-8 text file and processes it in request memory.
- Public responses are marked `no-store`; results have no server-side analysis identifier or
  history endpoint.
- SQLite persistence, external AI, host-connected sources, notes, settings, audit, retention,
  backup, and private report formats are unavailable in the public profile.
- Private HTTP and WebSocket routes return `404` from the public deployment.
- Browser-held evidence clears on explicit clear, refresh, tab close, or 30 minutes of inactivity.

Try the [session-only public demo](https://ca-tracehawk-public-demo.bluebush-2bdd630a.germanywestcentral.azurecontainerapps.io/).
Use only sanitized evaluation logs; do not upload production, client, credential, or confidential
data.

## Interactive Learning Tool

- Eight chapters explain Upload, Incidents, Findings, Evidence, Entities, MITRE ATT&CK, Reports,
  and the Detection Library.
- Each chapter documents what every panel shows, why it exists, how to interpret a worked example,
  and what its view-specific controls do.
- Contextual question buttons open the relevant chapter from the investigation workspace.
- Keyboard-accessible walkthroughs bind directly to stable controls in the real interface.
- Eight separate 60–90 second videos use an English male voice, visible click indicators, on-video
  narration text, and English WebVTT captions.
- Videos are self-hosted, lazy-loaded, analytics-free, and generated only from the fixed sanitized
  SSH sample. Visitor uploads are never used for recordings.

## Upgrade

No schema migration is required from v0.9.0. Back up `tracehawk.db`, update the application, and
start the private profile normally.

## Scope And Limitations

TraceHawk remains a single-replica local-first portfolio system. The public deployment is a
bounded demonstration and learning environment, not tenant-isolated storage, a production SIEM,
certified training, autonomous response, or evidence of production SOC adoption. Process-local
rate and concurrency controls require one public replica.
