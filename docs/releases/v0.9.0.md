# TraceHawk v0.9.0

TraceHawk v0.9.0 is the technical-excellence release. It closes evidence-integrity,
correlation-semantics, collector-bounding, external-validation, and frontend-contract gaps while
preserving the local-first single-instance scope.

## Evidence Integrity

- Every unpurged raw evidence line is SHA-256 verified before persistence.
- Counter, ownership, and graph references are rejected before an existing analysis is replaced.
- Live WebSocket snapshots carry process-local HMAC attestations that cover bounded-retention
  provenance; missing, modified, or previous-process snapshots fail before write.
- Alembic migration `0003_evidence_integrity` stores integrity status, origin, verified-line count,
  live attestation state, and retention counters.

The attestation proves that a saved payload matches a snapshot emitted by the current backend
process. It does not prove sensor identity or legal chain of custody.

## Correlation And Collection

- Rule correlation behavior, entity keys, strict time gaps, and sequence rationale are declarative.
- A versioned five-pattern library composes rule behaviors without hard-coded rule IDs.
- Incident grouping requires a bounded common entity across all members, preventing transitive
  A-user/B-IP/C bridge merges.
- Live file/interface analysis uses fixed raw/event windows and exposes total, retained, and
  dropped counters.
- The opt-in syslog service accepts bounded TCP or UDP lines on loopback by default with queue,
  connection, idle, line, batch, and failure-recovery limits.

## Evaluation

- All 66 YAML rules retain positive contract coverage with committed benign controls.
- IoT-23 evaluation separates development, validation, final, and negative capture roles in a
  frozen manifest.
- The final report covers 45,158 labeled rows across seven captures.
- The stable-endpoint retry rule remains a low-confidence hunting indicator: its final C2-indicator
  slice records 0.0870 precision and 0.6667 recall. This poor precision is retained visibly and is
  not a production C2-detector claim.

## Frontend And API Contract

- FastAPI OpenAPI and TypeScript declarations are generated deterministically and checked for drift
  in local, GitLab, and GitHub gates.
- Core browser response types come from the generated backend contract.
- Thirty-one behavior/axe tests enforce 70% lines/statements, 65% functions, and 50% branches across
  maintained frontend source.
- Five Chromium workflows cover demo/report, admin navigation, real-lab correlation, raw-evidence
  pivot, and rejected-action recovery.
- Coverage and browser reports are retained as CI artifacts.

## Upgrade

Back up `tracehawk.db`, then start v0.9.0 normally. Alembic upgrades a recognized v0.8.0 database
from `0002_case_integrity` to `0003_evidence_integrity`. Partial or unknown unversioned schemas stop
startup and require explicit recovery.

## Scope

TraceHawk remains a single-replica local-first portfolio system, not a multi-tenant SIEM. The
collector has no durable broker or replay protocol, live eviction loses old telemetry, generated
TypeScript is not runtime response validation, and external evaluation does not establish
production accuracy.
