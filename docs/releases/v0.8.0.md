# TraceHawk v0.8.0

TraceHawk v0.8.0 is the GitHub-readiness and evidence-integrity release. It closes persistence,
browser-test, release-provenance, and supply-chain gaps found during the final public review.

## Data Integrity

- Alembic refuses partial or unknown unversioned databases instead of stamping them as valid.
- SQLite foreign-key enforcement is enabled on every application connection.
- Re-analysis replaces earlier child records instead of retaining stale findings or incidents.
- Persisted cases retain source summaries, cross-source links, case quality, score breakdowns, and
  score rationale after reopening.

## Access And UI

- Host file, folder, Docker, and interface live sources require the admin role.
- Local rate limiting ignores spoofed deployment identity headers.
- Assistant settings work across the split local frontend through an explicit CORS `PUT` contract.
- The frontend consumes role metadata and hides host/admin controls from non-admin users.

## Verification And Supply Chain

- Playwright exercises a rendered Chromium investigation and report workflow.
- GitHub Actions are SHA-pinned and run weekly as well as on pushes and pull requests.
- Dependabot covers Python, npm, GitHub Actions, and Docker dependencies.
- Trivy fails CI on fixed HIGH or CRITICAL container vulnerabilities.
- Production Python is installed from a wheel and base images are digest-pinned.
- Fixed release timestamps and invariant PDF generation make committed reports reproducible.

## Public Export

- Release export refuses a dirty source tree.
- Explicit dirty previews identify themselves as `working-tree-preview` instead of claiming exact
  commit provenance.
- The public allowlist includes migrations, browser tests, pinned CI, and dependency automation.

## Upgrade

Back up `tracehawk.db`, then start v0.8.0 normally. Alembic upgrades the recognized v0.7.1 baseline
to `0002_case_integrity`. Unknown partial schemas stop startup and require explicit recovery.
