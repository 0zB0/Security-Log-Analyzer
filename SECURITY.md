# Security Policy

## Supported Version

Security fixes target the latest public release and `main`.

## Reporting A Vulnerability

Use GitHub private vulnerability reporting from the repository **Security** tab. Do not open a
public issue containing exploit details, credentials, private logs, or infrastructure information.

Include the affected version, reproduction steps, impact, and any proposed mitigation. Reports are
triaged before disclosure. No bug bounty or guaranteed response SLA is offered.

## Data Boundary

Use only sanitized logs and fixtures. Do not upload production credentials, client evidence,
private hostnames, internal addressing, session material, or personal data to the hosted demo.
